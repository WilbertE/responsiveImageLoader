jQuery.responsiveImageLoader

The problem
The problem with responsive websites is that they load one image file for desktop and mobile users. So mobile users have to load a large desktop image while it will be scaled down to a small size. This makes the page slow to load on mobile devices.

What this plugin does
With this plugin you can tell the browser which image to load. So mobile users load a small image and desktop image load a large image. When desktop users use a small window and resize it to a larger one the plugin will check if it needs to load a larger image file.

Setup
  1. Include jquery if you already haven't
  2. Include jquery.responsiveImageLoader.js
  3. Add the "responsive-image" class to images you want to handle
  4. Add html5 data attributes to the tag
    'data-ri-[minScreenWidth]-[src or background]="[filename]"'
Code Example
<img class='responsive-image'
     data-ri-0-src='images/turtle-small.jpg'
     data-ri-768-src='images/turtle-large.jpg'>

Image src demo:
- From 0 to 768 pixels small files will be loaded
- From 786 and higher large files will be loaded
